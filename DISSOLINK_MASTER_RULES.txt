SECTION 1 — WHAT IS DISSOLINK?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DissOLinK is the DISSOVERSE link routing system hosted at:
  https://links.dissoverse.com/[slug]

The domain links.dissoverse.com is CNAME'd to the GitHub Pages repo:
  https://github.com/analogflowers-coder/dissolinks

Each slug (e.g., /itsrockymontana, /ivydelicate, /mainsite, /shop) is a folder
containing an index.html file that performs a JavaScript redirect.

PURPOSE: Every bridge page serves one primary function — to FORCE users out of
in-app browsers (Instagram WebView, TikTok WebView, etc.) so that links open in
the device's native browser. This is critical for link.me and Shopify pages to
function correctly.

SECTION 2 — THE TWO LINK TYPES AND THEIR CORRECT CODE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TYPE A — ARTIST/EXTERNAL BRIDGE (e.g., itsrockymontana, ivydelicate)
  → Final destination: link.me/[artist-slug]
  → These are creator profile links that route to external link-in-bio pages
  → HOW IT WORKS: Bridge page → dissoverse.com/pages/go?url=link.me/[slug]
    → dv-bridge.liquid fires IAB escape → user lands on link.me in native browser
  → CRITICAL: link.me is WHITELISTED in dv-bridge.liquid (verified 2026-04-07)
  → DO NOT route directly to link.me — this skips IAB escape entirely

  CORRECT CODE TEMPLATE (TYPE A):
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ <!DOCTYPE html>                                                         │
  │ <html>                                                                  │
  │ <head><meta charset="utf-8"><title>Redirecting...</title></head>        │
  │ <body>                                                                  │
  │ <!-- BRIDGE ROUTED (INTERNAL). DO NOT CONVERT TO DIRECT REDIRECT.      │
  │      See BRIDGE_HARDLOCK.txt -->                                        │
  │ <script>                                                                │
  │ window.location.replace(                                                │
  │   "https://dissoverse.com/pages/go?url=https%3A%2F%2Flink.me%2F[SLUG]" │
  │ );                                                                      │
  │ </script>                                                               │
  │ <noscript>                                                              │
  │ <a href="https://dissoverse.com/pages/go?url=https%3A%2F%2Flink.me%2F[SLUG]"> │
  │ Click here</a></noscript>                                               │
  │ </body>                                                                 │
  │ </html>                                                                 │
  └─────────────────────────────────────────────────────────────────────────┘
  NOTE: Replace [SLUG] with the artist slug (e.g., itsrockymontana)
  NOTE: The final user destination IS link.me — dissoverse.com/pages/go is
        only an intermediate IAB escape handler. Users end up on link.me.

TYPE B — STORE/INTERNAL BRIDGE (e.g., mainsite, shop, shopstore)
  → Destination: dissoverse.com/pages/go (or dissoverse.com/pages/go?url=...)
  → These are DISSOVERSE store links that route to Shopify store pages

  CORRECT CODE TEMPLATE (TYPE B):
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ <!DOCTYPE html>                                                         │
  │ <html>                                                                  │
  │ <head><meta charset="utf-8"><title>Redirecting...</title></head>        │
  │ <body>                                                                  │
  │ <!-- BRIDGE ROUTED (INTERNAL). DO NOT CONVERT TO DIRECT REDIRECT.      │
  │      See BRIDGE_HARDLOCK.txt -->                                        │
  │ <script>                                                                │
  │ window.location.replace("https://dissoverse.com/pages/go");            │
  │ </script>                                                               │
  │ <noscript><a href="https://dissoverse.com/pages/go">Click here</a>     │
  │ </noscript>                                                             │
  │ </body>                                                                 │
  │ </html>                                                                 │
  └─────────────────────────────────────────────────────────────────────────┘

SECTION 4 — WHAT MUST NEVER CHANGE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ❌ NEVER route a TYPE A link directly to link.me without going through
     dissoverse.com/pages/go?url= first
     → Skipping /pages/go means the IAB escape never fires
     → Direct redirect to link.me = NO in-app browser escape = broken bridge

  ❌ NEVER use dissoverse.com/pages/go WITHOUT the ?url= parameter for
     TYPE A (artist) links
     → /pages/go with no parameter defaults to dissoverse.com homepage
     → This was the root cause of DISSOLINK-001 (12-hour outage)

  ❌ NEVER remove the BRIDGE ROUTED comment from any bridge page
     → The comment is a permanent marker that protects against accidental
       conversion to a simple direct redirect

  ❌ NEVER use dissoverse.com/pages/go (no param) as the destination for
     artist links — ALWAYS include ?url=https%3A%2F%2Flink.me%2F[slug]

  ❌ NEVER change any bridge page destination without:
     (1) Checking the Link Map in Section 3
     (2) Confirming the link type (TYPE A vs TYPE B)
     (3) Getting GRANDMASTER sign-off for any new slugs added

  ❌ NEVER assume "bridge = goes to dissoverse.com" — bridge format is used
     for BOTH TYPE A and TYPE B. The bridge is for in-app browser escape.
     The destination is determined by the link type, not the bridge format.

  ❌ NEVER deploy a DissOLinK change to only GitHub OR only desktop.
     Push Protocol is MANDATORY: GitHub + desktop hard copy at minimum.

ROOT CAUSE:
  Architecture documentation used the term "BRIDGE ROUTED (INTERNAL)" in the
  HTML comment, which was misread as "this link goes to an internal DISSOVERSE
  page." This caused confusion between the bridge FORMAT (for in-app escape)
  and the bridge DESTINATION (which can be either link.me OR dissoverse.com
  depending on link type).

  The "INTERNAL" in the comment refers to the ROUTING MECHANISM being internal
  to the DISSOVERSE link system — NOT the destination being dissoverse.com.

CONTRIBUTING FACTORS:
  1. Ambiguous comment: "BRIDGE ROUTED (INTERNAL)" — "INTERNAL" caused
     misinterpretation of the destination type
  2. No Link Map documented — no reference list of slugs and their types
  3. No governance file existed to prevent this class of error
  4. Autocorrect in Grandmaster's message ("dissonance" vs "dissoverse")
     added an additional layer of ambiguity
